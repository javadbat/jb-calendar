`JBCalendar` styling is theme-driven. General design tokens are documented in the core theme pages. The shared files in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) are reusable example theme packs built from those common token ideas, and component-specific style files live beside each component's stories.

## Common Design Tokens

Use the core theme docs as the source of truth for shared design-system variables:

- [Theme Colors](../../jb-core/theme/stories/colors.mdx) explains the general color palette variables, default colors, customization approaches, and color generation helpers.
- [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) explains shared size and radius variables.
- [Theme Custom Galleries](https://javadbat.github.io/design-system/?path=/docs/theme-custom-material--docs) gathers the shared custom style recipes across components.

Calendar recipes should use those common tokens where possible, then map them into `--jb-calendar-*` variables only where the calendar needs a component-specific value.

## Styling Model

Use three layers when styling calendar examples:

- Common design tokens are documented in [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx).
- Reusable example theme packs live in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and can be shared across components.
- Component recipe files live in [`modules/jb-calendar/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-calendar/stories/styles).
- Public component variables use the `--jb-calendar-*` prefix and are the stable styling API.

Each example uses one shared example theme pack from [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and one calendar-specific recipe from [`modules/jb-calendar/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-calendar/stories/styles). The style story applies both classes directly to `JBCalendar`.

## Recommended Customization Path

1. Use `--jb-calendar-*` variables first for component internals.
2. Use host styles for the outside surface: background, padding, shadow, border, and component-level border radius.
3. Use common theme variables from [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) when the style should align with the design system.
4. Put reusable theme-pack values in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) only when they should be shared by multiple component demos.
5. Add a new component variable only when a needed internal part cannot be customized with the current public variables.

Use `::part(...)` sparingly for exposed surfaces and state tokens that need styling beyond variables, such as `::part(disabled-day)`, `::part(today-day)`, or `::part(navigator)`. The Material, Fluent, Carbon, Candy, and Terminal recipes include small examples.

## Theme File Pattern

Use [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) as the common design-token guides. Theme-pack files should keep reusable demo variables in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles), then each component should keep its own style CSS in its story [`styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-calendar/stories/styles) folder.

```css
/* https://github.com/javadbat/design-system/blob/main/docs/styles/carbon.css */
.theme-name-style {
  /* shared design-system variables */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-calendar/stories/styles/style-carbon.css */
jb-calendar.theme-name-calendar {
  /* calendar-specific variables */
}
```

This keeps common theme tokens reusable and lets each component page show only the CSS it needs.
